# Project Protection Rules

CRITICAL:

- Never delete existing files.
- Never rename existing files.
- Never restructure the project.
- Never remove folders.
- Never overwrite package.json unless explicitly requested.
- Never modify Firebase configuration files unless explicitly requested.
- Before deleting any file, ask for user confirmation.

When making changes:
- Edit only the files required.
- Preserve all existing functionality.
- Ensure TypeScript compiles successfully.
- Ensure npm build passes before completing the task.
